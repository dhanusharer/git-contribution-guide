# 🧪 First Contribution Sample Repository

> **This repo exists solely so you can practice making your first open source contribution — with no pressure, no stakes, and no chance of breaking anything important!**

---

## 🎯 Your Mission (Should You Choose to Accept It)

Add your name to the [`contributors.md`](./contributors.md) file by following the steps below.

This is the classic beginner contribution — simple, safe, and genuinely useful. Every name you see in `contributors.md` belongs to someone who completed this exact exercise.

---

## ✅ How to Contribute

Follow these steps carefully. Detailed explanations of each step are in the [full guide](../README.md).

### 1. Fork This Repository

Click the **Fork** button in the top-right corner of this page on GitHub. This creates your own copy.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/first-contribution-guide.git
cd first-contribution-guide/sample-repo
```

### 3. Create a New Branch

```bash
git checkout -b docs/add-YOUR-NAME-to-contributors
```

Example:
```bash
git checkout -b docs/add-jane-doe-to-contributors
```

### 4. Edit contributors.md

Open `contributors.md` in your editor and add a row with your information:

```markdown
| Your Name | [@yourusername](https://github.com/yourusername) | your country/city | Added my name! |
```

### 5. Commit Your Change

```bash
git add contributors.md
git commit -m "docs: add YOUR NAME to contributors list"
```

### 6. Push and Open a Pull Request

```bash
git push origin docs/add-YOUR-NAME-to-contributors
```

Then go to your fork on GitHub and click **"Compare & pull request"**.

**PR Title:** `docs: add YOUR NAME to contributors list`

**PR Description:** Feel free to say hi! Tell us where you're from, what brought you to open source, or anything else.

---

## 📋 Contribution Rules

To keep this practice repo friendly and functional:

1. **One row per person** — check if your name already appears before adding
2. **Only edit `contributors.md`** — PRs changing other files will be closed
3. **Use the correct format** — match the table structure exactly
4. **Be kind in comments** — this is a learning environment
